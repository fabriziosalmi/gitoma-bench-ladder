# Contributing to gitoma-bench-ladder

This document outlines the guidelines and expectations for contributing to the gitoma-bench-ladder project.

## Overview

We welcome contributions from the community! Whether you are a new contributor or an experienced engineer, your input is valuable. This repository focuses on benchmarking and testing related logic.

## Code of Conduct

We are committed to maintaining a friendly and inclusive development environment. Please read our [CODE_OF_CONDUCT.md] (if one exists, otherwise mention a standard one) for details on our Code of Conduct.

## How to Contribute

1. **Fork the Repository**: Fork this repository to your own account.
2. **Clone Your Fork**: Clone your fork locally.
3. **Create a Feature Branch**: Create a new branch for your changes (e.g., `feature/new-benchmark` or `fix/bug-name`).
4. **Make Your Changes**: Implement your changes, ensuring they adhere to the project's standards.
5. **Tests and Validation**: Ensure that any new or modified logic is accompanied by appropriate unit tests (see `tests/` directory) to validate correctness.
6. **Commit**: Commit your changes with a clear, descriptive message following Conventional Commits standards.
7. **Pull Request (PR)**: Open a Pull Request targeting the main branch of the repository.

## Guidelines

* **Code Style**: Please adhere to Python best practices (PEP 8). Use linters (like Flake8) to maintain code quality.
* **Testing**: All new features or significant changes must include corresponding tests. Tests should cover edge cases.
* **Documentation**: If you introduce new functions, classes, or complex logic, please add docstrings explaining what the code does, its parameters, and what it returns.
* **Issue Tracking**: Report bugs or suggest features via the Issues tracker. Please provide clear reproduction steps and expected vs. actual behavior.

## Development Workflow

We use Git for version control. Please ensure your branch is up-to-date with the latest changes from the main branch before submitting a PR.

## Reporting Issues

If you encounter bugs, please first check if the issue has already been reported. If not, open a new issue detailing:

- **What** is happening.
- **Where** it happens (including relevant input/setup).
- **Expected Behavior**.
- **Actual Behavior**.

Thank you for considering contributing to gitoma-bench-ladder!
