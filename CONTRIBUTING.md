# Contributing Guidelines

## Getting Started

1. Fork the repository on GitHub.
2. Clone your forked repository to your local machine:

   ```bash
   git clone https://github.com/YOUR_USERNAME/gitoma-bench-ladder.git
   ```

3. Create a new branch for your feature or bugfix:

   ```bash
   git checkout -b feature-name
   ```

## Making Changes

- Ensure you have Python installed (version 3.8 or higher).
- Install the project dependencies using pip:

   ```bash
   pip install -r requirements.txt
   ```

- Make your changes in the `src/` directory.
- Add new features or modify existing ones following the project's coding standards.

## Testing Changes

- Run the tests to ensure your changes don't break existing functionality:

   ```bash
   python -m pytest tests/
   ```

## Submitting Changes

1. Commit your changes with a clear and concise commit message.
2. Push your changes to your forked repository:

   ```bash
   git push origin feature-name
   ```
3. Open a pull request on the original repository's GitHub page.
4. Wait for feedback and make any necessary adjustments.

## Code of Conduct

Please follow the project's code of conduct when contributing. We welcome all contributions and are committed to creating a welcoming environment for everyone.
