# Vulnerability Disclosure Policy

If you discover a security vulnerability in this project, please follow these steps:

1. Verify the vulnerability exists using the latest version of the code.
2. Do not disclose the vulnerability publicly or contact anyone outside the project maintainers.
3. Email the project maintainer at [maintainer@example.com](mailto:maintainer@example.com) with details of the vulnerability.
4. Provide as much information as possible to help reproduce the issue, including steps to replicate it and any error messages.

We appreciate your responsible disclosure and will address all valid vulnerabilities promptly.
