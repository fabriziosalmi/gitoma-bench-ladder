## Vulnerability Disclosure Policy

We take security seriously and encourage responsible disclosure of vulnerabilities. If you discover a vulnerability in this project, please follow these steps:

1. Verify the vulnerability exists and is not already fixed in the latest version.
2. Contact the project maintainers through [email address or issue tracker].
3. Provide detailed information about the vulnerability, including steps to reproduce it.
4. Do not publish or share any details of the vulnerability publicly until it has been addressed.

Once we receive a valid report, we will:
- Acknowledge receipt of the report.
- Investigate and validate the vulnerability.
- Provide an update on our progress within [X] business days.
- Disclose the vulnerability and provide a fix as appropriate.

Thank you for helping us improve the security of this project.
