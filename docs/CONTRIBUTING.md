# Contributing to gitoma-bench-ladder

Welcome to the gitoma-bench-ladder project! We appreciate your contributions.

## Code of Conduct

We expect all contributors to adhere to our [CODE OF CONDUCT](CODEOFCONDUCT.md) (if applicable, otherwise mention it).

## Development Workflow

1. **Fork** the repository.
2. **Clone** your fork locally: `git clone <your-fork-url>`. 
3. **Create a new branch** for your feature or fix: `git checkout -b feature/my-new-feature`.
4. **Make your changes** and ensure tests pass.
5. **Commit** your changes with a descriptive message: `git commit -m 'feat: implement new feature'`. 
6. **Push** to your fork: `git push origin feature/my-new-feature`.
7. **Open a Pull Request (PR)** against the main branch of the repository.

## Code Style

We follow idiomatic Go style. Please ensure your code adheres to the standards outlined in `CONTRIBUTING.md` (or other style guides).

## Testing

All relevant components should have corresponding unit tests. Please ensure new features are tested thoroughly.

## Reporting Issues

Please use the GitHub Issues tracker to report bugs or suggest features.
