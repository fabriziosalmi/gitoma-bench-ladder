## Development Workflow

1. Fork the repository.
2. Create a new feature branch (`gitoma/feature-name`).
3. Make your changes and ensure tests pass.
4. Commit changes with descriptive messages.
5. Open a Pull Request.

## Benchmarking Guidelines

When adding new benchmarks or modifying existing ones, ensure they are: 

- Reproducible.
- Clearly documented regarding setup and expected performance metrics.
- Follow the established benchmarking conventions.

## Reporting Issues

Please use the GitHub Issues tracker to report bugs or suggest features.
